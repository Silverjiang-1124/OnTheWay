# OnTheWay 迭代清单

## 本轮目标

把项目从“功能可用”提升到“出发前可靠可用”：修复 lint 阻塞、避免用户数据丢失、收敛 HTML 安全风险，并补齐打包重量、备份导出、打印/PDF 等核心户外使用场景。

## 已实施

- 基础质量：修复 Hooks 条件调用、`any` 类型、Fast Refresh 文件结构和表单初始化方式。
- 数据安全：store 迁移改为保留已有装备/行程并补齐字段；新增 JSON 备份导出和合并导入。
- HTML 安全：行程计划展示前移除脚本、危险标签、事件属性和 `javascript:` 链接；不再使用 iframe，改为页面内安全渲染。
- 打包清单：新增总重量、已打包重量、未打包重量、完成度、按分类重量、按负责人重量。
- 导出/打印：复制摘要包含风险信息和重量；打印隐藏导航与操作控件，保留行程概要、计划、清单、游记。
- 移动端：详情页操作按钮支持换行，弹窗和清单操作适配窄屏。
- 行程风险：新增天气、补给、交通、紧急联系人、注意事项字段。
- 装备重量：补齐主要 seed 装备重量，新增 `3L水袋（满水）`，并让启动迁移自动为老数据补缺失字段和新增 seed 装备。

## 后续可迭代

- 将行程计划从 HTML 片段迁移到 Markdown 或结构化日程编辑器。
- 增加装备“存放位置、购买日期、保质期/电量状态、是否消耗品”等字段。
- 支持按天拆分补给和水量，并自动估算人均负重。
- 增加“水/燃料/食物”等消耗品标记，区分出发重量、当前重量和基础重量。
- 增加 PWA 离线安装和本地文件备份提醒。
- 增加 GPX 文件上传、轨迹文件归档和离线地图链接管理。

## 验收清单

- `npm run lint` 通过。
- `npm run build` 通过。
- 新增、编辑、删除装备后，已有关联行程清单同步正确。
- 旧 localStorage 数据升级后，用户装备不被 seed 数据覆盖。
- 行程详情不存在、切换详情页、编辑游记、切换打包状态时无 Hooks 报错。
- 打印/PDF 中不出现侧边栏、底部导航、编辑/删除/导出按钮。
- JSON 导出后清空浏览器数据，再导入能恢复装备、行程、打包状态和游记。
